NAVI Herramientas & Activos — Entrega técnica MVP

1. Resumen ejecutivo

El MVP de NAVI Herramientas & Activos permite gestionar herramientas, activos de taller y su hoja de vida técnica, integrando inventario, documentos, mantenimientos, accesorios, prácticas seguras, trazabilidad y exportación de información.

El sistema fue construido como una solución web basada en .NET 8, con API REST, panel administrativo en Blazor Server, almacenamiento documental en MinIO y persistencia en SQL Server.

El objetivo principal del MVP es demostrar el flujo funcional completo para administrar herramientas de taller, tomando como referencia el comportamiento esperado en Dynamics/Fenix365 y adaptándolo a un módulo especializado para control operativo, técnico y documental.


2. Stack tecnológico

Backend: .NET 8 Web API
Frontend administrativo: Blazor Server
Base de datos: SQL Server
ORM: Entity Framework Core
Documentos: MinIO
Exportación Excel: ClosedXML
Jobs / procesos futuros: Hangfire
Documentación API: Swagger
Control de versiones: Git / GitHub

5. Módulos funcionales implementados

5.1 Inventario maestro

Funcionalidades:

- Consulta general de herramientas y activos.
- Búsqueda por código, nombre, serial, sede, tipo o categoría.
- Filtro por estado operativo.
- Filtro por sede.
- Semáforos visuales de estado.
- Filtros rápidos por estado.
- Acceso a detalle de herramienta.
- Acceso a hoja de vida técnica.

Ruta Admin:

/tools

Endpoint principal:

GET /api/tools


5.2 Detalle de herramienta

Funcionalidades:

- Consulta de información general.
- Cambio de estado operativo.
- Marcación como herramienta especializada o no especializada.
- Registro de cambios en trazabilidad.

Ruta Admin:

/tools/{toolId}


5.3 Hoja de vida técnica

Funcionalidades:

- Información general del equipo.
- Especificaciones técnicas.
- Garantía.
- Vida útil.
- Plan de mantenimiento.
- Accesorios.
- Registro documental.
- Cronograma de mantenimiento.
- Prácticas seguras.
- Eventos de hoja de vida.
- Alerta visual de mantenimiento.
- Accesos rápidos a gestión.
- Exportación PDF mediante impresión.
- Exportación Excel .xlsx.

Ruta Admin:

/tools/{toolId}/technical-life-record

Endpoints:

GET /api/tools/{id}/technical-life-record
GET /api/tools/by-code/{internalCode}/technical-life-record
GET /api/tools/{id}/technical-life-record/export-excel
GET /api/tools/by-code/{internalCode}/technical-life-record/export-excel


5.4 Gestión de datos técnicos

Funcionalidades:

- Edición de marca, modelo, serial, activo fijo y código Fenix365.
- Edición de voltaje, capacidad, proveedor, garantía y vida útil.
- Actualización de fechas de mantenimiento.
- Registro de evento técnico.

Ruta Admin:

/tools/{toolId}/technical-life-record/edit

Endpoints:

PATCH /api/tools/{id}/technical-data
PATCH /api/tools/by-code/{internalCode}/technical-data


5.5 Gestión documental

Funcionalidades:

- Carga de documentos.
- Clasificación por tipo.
- Descarga de documentos.
- Eliminación lógica de documentos.
- Limpieza automática de evidencia cuando un documento eliminado estaba asociado a un mantenimiento.
- Registro de eventos DocumentUploaded, DocumentDeleted y MaintenanceEvidenceDocumentCleared.

Ruta Admin:

/tools/{toolId}/documents

Endpoints:

GET /api/tools/{id}/documents
GET /api/tools/by-code/{internalCode}/documents
POST /api/tools/{id}/documents
POST /api/tools/by-code/{internalCode}/documents
GET /api/tools/documents/{documentId}/download
DELETE /api/tools/documents/{documentId}

14. Conclusión

El MVP de NAVI Herramientas & Activos queda funcional para validar el flujo completo de administración de herramientas y activos de taller.

La solución permite consultar inventario, administrar hoja de vida técnica, cargar y descargar documentos, asociar evidencias a mantenimientos, gestionar accesorios, prácticas seguras y visualizar trazabilidad completa de eventos.

El sistema queda preparado como base para evolucionar hacia una solución integrada con Dynamics/Fenix365 y con capacidades operativas para escritorio y móvil.
